# Obsidian 자동 갱신 전략

RAG 시스템이 사용자와의 질의응답(Q&A)에서 얻은 새로운 지식을 [[Obsidian]] 노트에 자동으로 반영하는 전략을 다룬다. 핵심은 **사용자 승인** 을 거치는 것이다. #obsidian #automation

## 왜 자동 갱신인가

[[RAG 시스템 설계]] 의 검색 품질은 결국 볼트에 담긴 지식의 최신성에 달려 있다. Q&A 과정에서 새로 확인된 사실이 노트로 되돌아오지 않으면, 같은 질문을 다시 받았을 때도 시스템은 여전히 옛 정보를 검색한다. 그래서 답변 생성 후 학습된 내용을 다시 노트로 환류(feedback)하는 루프가 필요하다. #rag

## 승인 기반 갱신 흐름

무분별한 자동 수정은 볼트를 오염시킬 수 있으므로, 항상 사람이 최종 확인한다.

1. 질의응답에서 노트에 반영할 만한 후보 지식을 추출한다.
2. 시스템이 초안(diff)을 만들어 `90_Generated/` 폴더에 제안한다.
3. 사용자가 초안을 검토하고 승인/수정/거절한다.
4. 승인된 내용만 원본 노트에 병합하고 `updated` 필드를 갱신한다.

### 충돌 방지 규칙

- 프론트매터의 `title` 과 기존 [[wikilink]] 구조는 절대 임의로 바꾸지 않는다.
- 기존 문장을 삭제하기보다 새 섹션을 추가하는 방향을 우선한다.
- 자동 생성분에는 항상 출처(질문 ID)를 주석으로 남긴다. #obsidian

## 기대 효과

지식이 축적될수록 검색 문맥이 풍부해지고, 사용자는 반복 질문을 줄일 수 있다.
