# OpenAI API

이 프로젝트에서 OpenAI API는 두 가지 역할을 한다. 텍스트를 벡터로 바꾸는 **임베딩** 과, 검색된 문맥으로 답변을 만드는 **채팅(생성)** 이다. #openai #llm

## 임베딩

- 모델: `text-embedding-3-small` 을 기본으로 사용한다. 비용 대비 품질이 좋고 1536차원 벡터를 반환한다.
- 용도: 노트 청크와 사용자 질문을 같은 임베딩 공간에 놓아 벡터 검색을 가능하게 한다.
- 이 벡터는 [[Elasticsearch]] 의 `dense_vector` 필드에 저장된다. #embedding

### 요청 흐름

`텍스트 → 임베딩 API → 1536차원 벡터 → 색인/검색`

## 채팅(생성) 모델

검색된 문맥을 시스템 프롬프트에 담아 채팅 모델에 전달하면, 근거에 기반한 답변을 생성한다. 전체 파이프라인에서의 위치는 [[RAG 시스템 설계]] 를 참고한다.

## Mock Provider로 대체

개발이나 데모 단계에서는 실제 API 호출 없이 동작을 검증하고 싶을 때가 많다.

- 임베딩 대신 결정적(deterministic)인 가짜 벡터를 반환하는 Mock을 끼운다.
- 채팅 대신 고정된 템플릿 응답을 돌려준다.
- 인터페이스를 동일하게 유지하면, 실제 OpenAI 구현과 Mock 구현을 설정만 바꿔 교체할 수 있다.

이렇게 하면 네트워크나 비용 걱정 없이 [[RAG 시스템 설계]] 의 흐름 전체를 로컬에서 시험할 수 있다. #llm
