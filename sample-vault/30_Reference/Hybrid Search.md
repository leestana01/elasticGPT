# Hybrid Search

하이브리드 검색은 키워드 기반 검색과 벡터 기반 검색을 결합해 각 방식의 약점을 서로 보완하는 검색 전략이다. #search #hybrid

## 두 검색의 강점과 약점

- **키워드 검색(BM25)**: 정확한 단어 일치, 고유명사, 코드 식별자에 강하다. 하지만 표현이 다르면 놓친다.
- **벡터 검색(kNN)**: 의미가 비슷한 문서를 잡아내 동의어와 문맥에 강하다. 하지만 드문 고유명사에는 약할 수 있다.

두 방식은 실패 지점이 다르므로 함께 쓰면 재현율(recall)이 올라간다. 두 검색 모두 [[Elasticsearch]] 안에서 실행할 수 있다. #retrieval

## Reciprocal Rank Fusion

서로 다른 두 결과 목록을 하나로 합치는 대표적 방법이 상호 순위 융합(Reciprocal Rank Fusion, RRF)이다.

- 각 문서에 대해 두 목록에서의 **순위(rank)** 만 사용한다. 점수 스케일이 달라도 안전하게 합쳐진다.
- 문서 d의 점수는 각 목록에서 `1 / (k + rank_d)` 를 더한 값으로 계산한다. (k는 보통 60 정도의 상수)
- 두 목록 모두에서 상위에 오른 문서가 자연스럽게 최상단으로 올라온다.

### 순서 정리

`BM25 결과 + 벡터 결과 → RRF 융합 → 상위 문맥 선택`

이렇게 뽑은 문맥이 답변 생성의 입력이 된다. 전체 흐름은 [[RAG 시스템 설계]] 를 참고한다. #search
